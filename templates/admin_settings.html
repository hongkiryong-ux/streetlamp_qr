Resend (Render에서 SMTP 타임아웃일 때 권장)

Render 무료 플랜 등에서 smtp.gmail.com 로 나가는 SMTP가 막히거나 계속 타임아웃이면, resend.com 에서 API 키를 발급하고 Environment에 RESEND_API_KEY=re_... 를 넣습니다(HTTPS, 포트 443). 발신: RESEND_FROM 을 비우면 기본 onboarding@resend.dev만 사용합니다. (@gmail.com 은 Resend에서 발신 주소로 쓸 수 없습니다. 도메인을 Resend에 등록·검증한 뒤 noreply@내도메인 을 RESEND_FROM에 넣으세요.) 고급: 검증된 발신만 SMTP_FROM과 같게 쓰려면 RESEND_USE_SMTP_FROM=true. 테스트는 Resend 정책에 맞는 받는 주소로 하세요.

SMTP (Resend 미사용 시)

Render → Environment 예: SMTP_HOST=smtp.gmail.com, SMTP_PORT=587(또는 타임아웃 시 465), SMTP_USER, SMTP_PASSWORD(앱 비밀번호), SMTP_FROM, SMTP_USE_TLS=true · 선택: SMTP_TIMEOUT=120(초), 587이 안 되면 SMTP_PORT=465 + SMTP_USE_SSL=true (Gmail SMTPS)

회사 메일은 보안 정책상 발송이 막힐 수 있습니다. 그 경우 SMTP 대신 다른 서비스 검토가 필요합니다.

발송 성공 시 위 회색 안내에 수신 주소가 표시됩니다. 메일이 없으면 그 주소의 스팸함을 확인하고, Render Logs에서 Resend 사용 시 [resend] ok, SMTP 사용 시 [smtp] sendmail ok 로그를 확인하세요.
